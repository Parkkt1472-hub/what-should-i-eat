# 프로젝트 요약: 오늘 뭐 먹지?

## 프로젝트 개요

**오늘 뭐 먹지?**는 음식 선택 고민을 즉시 해결해주는 프로덕션 레디 웹 앱 MVP입니다. 혼자, 커플, 가족, 친구와 함께 먹을 음식을 빠르게 추천받을 수 있으며, 만들어 먹기, 배달, 외식 등 다양한 상황에 맞는 결정을 제공합니다.

## 구현된 핵심 기능

### 1. 사용자 플로우

**홈 화면**에서는 중앙에 큰 CTA 버튼 "🍽 오늘 뭐 먹지?"가 배치되어 있으며, 하단에는 "정확도 높이기" 버튼이 제공됩니다. 사용자는 클릭 한 번으로 즉시 결정 프로세스를 시작할 수 있습니다.

**결정 플로우**는 두 단계로 구성됩니다. 첫 번째 단계에서는 "누가 먹나요?"를 선택합니다 (나 혼자, 커플, 가족, 친구). 두 번째 단계에서는 "어떻게 먹을까요?"를 선택합니다 (만들어 먹기, 배달, 외식). 외식을 선택한 경우 추가로 장소 선택 단계가 나타납니다 (근처 간단 외식, 가까운 시내, 기분전환 야외).

**결과 화면**에서는 추천된 메뉴와 이유가 표시되며, 선택한 방식에 따라 다양한 액션 버튼이 제공됩니다.

### 2. 만들어 먹기 모드

만들어 먹기를 선택하면 메뉴 이름, 추천 이유, 필요한 재료 목록이 표시됩니다. 사용자는 다음 액션을 수행할 수 있습니다.

**레시피 보기** 버튼은 Google 검색으로 연결되어 해당 메뉴의 레시피를 찾을 수 있습니다. **유튜브로 배우기** 버튼은 YouTube 검색으로 연결되어 동영상 레시피를 시청할 수 있습니다. **토스쇼핑에서 재료 구매** 버튼은 Toss Shopping으로 연결되어 필요한 재료를 바로 구매할 수 있습니다.

### 3. 배달 모드

배달을 선택하면 추천 메뉴와 이유가 표시되며, **배달 검색하기** 버튼을 통해 Google Maps에서 근처 배달 가능한 식당을 찾을 수 있습니다.

### 4. 외식 모드

외식을 선택하면 장소 선택에 따라 검색 쿼리가 달라집니다. **근처 간단 외식**은 "{메뉴} near me"로 검색하고, **가까운 시내**는 "{메뉴} 맛집"으로 검색하며, **기분전환 야외**는 "전망 좋은 {메뉴} 맛집"으로 검색합니다. 모든 검색은 Google Maps로 연결됩니다.

### 5. 가족 모드 로직

가족을 선택하면 시스템은 자동으로 아이들에게 적합한 메뉴만 필터링합니다. 매운 음식(spicyLevel > 1)은 제외되며, familyFriendly가 true인 메뉴만 추천됩니다. 추천 이유도 가족 맞춤형으로 변경됩니다 (예: "아이들도 좋아하는 메뉴예요").

### 6. 메뉴 데이터베이스

35개 이상의 다양한 메뉴가 데이터베이스에 포함되어 있습니다. 한식(김치찌개, 된장찌개, 불고기, 제육볶음, 비빔밥 등), 중식(짜장면, 짬뽕, 탕수육, 마파두부 등), 일식(초밥, 라멘, 돈카츠, 우동 등), 양식(파스타, 피자, 스테이크 등), 분식(떡볶이, 김밥, 라면 등)이 포함됩니다.

각 메뉴는 카테고리, 재료 목록, 가족 친화성, 매운 정도, 난이도 등의 메타데이터를 가지고 있습니다.

## 기술 구현

### 프론트엔드 아키텍처

**Next.js 16.1.6 App Router**를 사용하여 최신 React 서버 컴포넌트 패턴을 적용했습니다. **TypeScript**로 타입 안정성을 확보했으며, **Tailwind CSS**로 모바일 우선 반응형 디자인을 구현했습니다.

### 상태 관리

React Hooks(useState)를 사용하여 간단하고 효율적인 상태 관리를 구현했습니다. 화면 전환(home, decision, result), 사용량 제한 모달 표시, 결정 데이터 저장 등의 상태를 관리합니다.

### 결정 엔진

규칙 기반 추천 시스템을 구현했습니다. 사용자 선택(누가, 어떻게)에 따라 메뉴를 필터링하고, 랜덤으로 선택하여 추천합니다. 컨텍스트 기반 이유 생성 기능이 포함되어 있으며, 외부 서비스 URL 자동 생성 기능도 제공됩니다.

향후 OpenAI API 통합을 위한 구조가 준비되어 있습니다.

### 사용량 제한

LocalStorage를 사용하여 일일 사용량을 추적합니다. 하루 3회 무료 제한이 적용되며, 한도 초과 시 프리미엄 모달이 표시됩니다. 날짜별 키(`usage_YYYY-MM-DD`)로 사용량을 저장합니다.

### UI/UX 디자인

모바일 우선 반응형 디자인으로 모든 디바이스에서 최적화된 경험을 제공합니다. 음식 테마 색상(오렌지-앰버-옐로우 그라데이션)을 사용했으며, 부드러운 트랜지션과 호버 효과를 적용했습니다. 큰 터치 영역과 명확한 CTA 버튼으로 사용성을 극대화했습니다.

## 수익화 구조

### 무료 플랜

하루 3회 무료 추천을 제공하며, 로그인이 필요 없습니다. 모든 핵심 기능을 사용할 수 있습니다.

### 프리미엄 플랜 (준비 중)

월 9,900원의 프리미엄 플랜은 다음 기능을 제공합니다. 무제한 결정, 가족 식단 자동 생성, 주간 메뉴 추천, 개인 취향 학습, 알레르기 필터링 등이 포함됩니다.

### Stripe 통합 준비

`lib/usageLimit.ts`에 Stripe 통합을 위한 placeholder 함수가 구현되어 있습니다. `createCheckoutSession` 함수는 향후 실제 Stripe API와 연동할 수 있도록 설계되었습니다. 구독 플랜 데이터 구조가 정의되어 있으며, 환경 변수 설정만으로 빠르게 활성화할 수 있습니다.

## 배포 준비

- ✅ 반응형 디자인 구현

- ✅ 외부 서비스 연동
- ✅ 테스트 완료

### 권장 배포 플랫폼

**Vercel**이 가장 권장됩니다. Next.js 제작사로 자동 최적화와 간편한 배포를 제공합니다. GitHub 연동만으로 자동 배포가 가능하며, 무료 플랜으로도 충분합니다.

**Netlify**도 좋은 대안입니다. 간단한 설정으로 배포 가능하며, 자동 SSL 인증서를 제공합니다.

**AWS Amplify**는 AWS 생태계와 통합이 필요한 경우 적합합니다.

**Docker**를 통한 자체 호스팅도 가능합니다. Dockerfile과 배포 가이드가 준비되어 있습니다.

## 향후 개선 방향

### AI 통합

OpenAI API를 통합하여 더욱 정교한 추천을 제공할 수 있습니다. 사용자의 과거 선택을 학습하고, 날씨, 시간대, 계절 등을 고려한 추천이 가능합니다. 자연어 이유 생성으로 더욱 설득력 있는 추천을 제공할 수 있습니다.

### 사용자 프로필

회원가입 및 로그인 기능을 추가하여 개인 취향을 저장하고, 알레르기 및 식이 제한을 설정할 수 있습니다. 식단 기록 및 분석 기능도 제공할 수 있습니다.

### 소셜 기능

친구와 메뉴 공유 기능, 가족 그룹 식단 관리, 리뷰 및 평점 시스템 등을 추가할 수 있습니다.

### 데이터 확장

더 많은 메뉴 추가(500+), 지역별 특산 음식, 계절 메뉴 추천, 영양 정보 제공 등을 구현할 수 있습니다.

## 실행 방법

npm install
npm run dev
